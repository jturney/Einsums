# ComputeGraph performance & feature audit (2026-07-14, pr11 tip f934a21b)

Four-way audit of the module (~27k lines): graph core + capture, executors + memory passes, optimizer passes, dispatch + cost model + IO.
Every item below is transparent to end users: no public API changes, no visible behavior changes (numerics identical; only speed, memory, or internal capability).
Line numbers refer to the pr11 worktree at the audit date.

## Tier 0 - correctness-adjacent (fix before building on top)

### A. ScaleAbsorption desyncs `desc->c_prefactor` from `params->c_pf` (CPU/GPU divergence)
`src/Passes/ScaleAbsorption.cpp:20-52,105-110`.
The CPU einsum executor reads prefactors live from `shared_ptr<EinsumParams>` (`Operations.hpp:3919`), while GPU dispatch reads the descriptor (`src/Graph.cpp:168-176`). ScaleAbsorption edits only `desc->c_prefactor`:
- On CPU the edit is invisible; the composed `scale(); einsum();` still runs the (dead, overwritten) `blas::scal` every execute - no arithmetic saved, just node-count cosmetics.
- The descriptor edit makes every later descriptor-reading pass misclassify the node as accumulating (nonzero `c_prefactor`): disables CSE eligibility (`CSE.cpp:88`), ContractionPlanning gates, and makes LoopInvariantHoisting treat it as self-modifying.
- On GPU, `desc.c_prefactor = scale_factor` while the scale executor lives only in the discarded CPU composite -> numeric divergence for absorbed scales.

Fix: for the overwrite case (`c_pf == 0`), the preceding Scale is dead - delete it outright (no composite, no descriptor edit). To genuinely absorb a scale, fold into `ab_pf` via `Graph::update_prefactors()` which syncs both `params` and `desc` (`Graph.cpp:954`).

### B. Residual bug-999 hole: `redirect_slot` is a pointer snapshot, not an indirection
`CSE.cpp:248-251` -> `Graph.hpp:1033-1040` (`from_slot->ptr = to_slot->ptr`).
This fixed the original baked-lambda class, but the copy goes stale: CSE runs at pipeline step 4, Materialization at step 12 (`Optimizer.cpp:201,222`). If the CSE survivor is a deferred `declare_tensor` output, its slot is null/placeholder at CSE time; after Materialization repoints the survivor's slot, the redirected duplicate's slot still holds the stale address. Same failure after `graph.rebind()` or a GPU shadow-swap.
Fix options (any is transparent): store `redirect_to` TensorId on the dead slot and resolve lazily at execute/rebind/materialize; or re-apply redirects after Materialization; or restrict CSE merges to survivors already materialized.

### C. `update_prefactors` matches `_params_store` by "count Einsum nodes before this one"
`src/Graph.cpp:954-996` (positional heuristic at :983-987; also 3 full `_nodes` scans and a dead loop at :964-970).
Any pass that reorders/removes einsum nodes (Reorder, CSE, DNE - all default) silently breaks the ordinal, so post-optimize prefactor updates can hit the wrong einsum.
Fix: store the owning `shared_ptr<EinsumParams>` on `EinsumDescriptor` at capture (it already holds `shared_ptr<EinsumIndices>`); update via the descriptor - O(1) and reorder-proof.

### D. ODR hazard: two different `ParsedEinsumSpec` structs, same fully-qualified name
`ComputeGraph/EinsumSpec.hpp:51-64` (with `conj_a/conj_b` + methods) vs `ComputeGraphTypes/EinsumSpec.hpp:19-24` (without). Any TU including both aggregate headers has an ODR violation with different layouts.
Fix: delete the ComputeGraphTypes copy (include/forward the real one) or rename it.

### E. `create_tensor` intermediates invisible to memory passes
`Graph.hpp:596-620` sets `is_intermediate=true` but never `release_fn`; FreeInsertion skips anything without one (`FreeInsertion.cpp:132`), and the Dataflow memory-budget gate keys off Materialize/Free nodes. Eager scratch tensors - the largest class in hand-written graphs - are never freed or budgeted.
Fix: populate a real `release_fn`/`materialize_fn` for eager intermediates (or route them through the deferred path internally).

### F. `redirect_slot` + later `rebind()` staleness (generalization of B)
`Graph.cpp:1033-1040`. A rebind of the survivor updates only its own slot; merged-away consumers keep the old buffer. Covered by the same indirection fix as B.

## Tier 1 - the big performance theme: topology is cached, nothing else is

### 1. `topological_sort()` has no `_sorted` early-out (top single win)
`src/Graph.cpp:749`. Only empty graphs early-return; 17 pass files call `graph.topological_sort()` at entry, so the default ~20-pass pipeline re-sorts an unchanged graph ~20x. Each sort calls `effective_io()` twice per node and rebuilds `_deps` in a second full pass (:855-889) duplicating the adjacency logic.
Fix: honor `_sorted` (+ a `_deps_valid` bit so `mark_sorted()` doesn't fake a valid `_deps`). `add_node` already clears `_sorted` (:554). Removes ~19/20 sorts with no behavior change.

### 2. `effective_io()` heap-copies every node's ID vectors; rebuilds ptr->tid map per control-flow node
`src/Graph.cpp:642-747`. Ordinary nodes (vast majority) return copies of `node.inputs/outputs` (2 allocations/node/call, ~40 per node per optimize). Loop/Conditional nodes rebuild a full `_tensors` scan (:706-710) on every call.
Fix: return const refs/span for the common branch; hoist ptr->tid map to one build per sort; cache subtree-augmented IO for control-flow nodes with invalidation.

### 3. No compiled execution plan for replay (umbrella)
Serial `Graph::execute()` (`Graph.cpp:1472-1670`):
- fmt::format zone names + per-input/output dim annotation strings rebuilt per node per execute (:1505-1556); `profile::push` has no is-recording fast path.
- `_timing_report` repopulated per execute with per-node label string copies, no reserve, even if never read (:1501,:1667-1670).
- full slot-map pointer-validation scan per execute (:1479).

`DataflowExecutor::execute()` (`src/Executor.cpp:136-229`): rebuilds the entire task scaffold per execute - per-node pred-handle vectors, `when_all` shared states (2 atomics + results vector each), 1-2 std::function closures, label concatenations (`label + "/start"`); sink detection re-scanned (:224-228).

`OpenMPExecutor` (`src/Executor.cpp:80-126`): BFS levelization recomputed per execute; separate `#pragma omp parallel for` fork/join barrier per level.

Fix: a per-sorted-generation compiled plan (invalidated exactly where `_sorted` is cleared): cached zone/annotation strings, flat predecessor index arrays, preallocated handle storage with resettable shared states, cached level partition and sink list. This is "execute the same optimized graph N times with zero re-analysis" - the SCF/CC replay case.

### 4. TaskPool submission storm
`src/Executor.cpp:184,216,219` -> `TaskPool.cpp:235-249`. Main thread is not a worker, so every submitted node takes the single `_external_mutex` and fires `notify_all()` (wakes all P workers per node).
Fix: batch-enqueue under one lock + one notify; or `notify_one` per task; or seed only dependency-free roots externally and let continuations self-enqueue on worker deques (that branch is already lock-free).

### 5. Per-execute einsum overhead despite capture-time caches
- `Operations.hpp:3918`: executor rebuilds `ParsedEinsumSpec parsed_live` copying 3 `vector<string>` per call, and omits the cached `indices->link_indices` -
- `StringDispatch.hpp:302-311` then recomputes link indices from scratch per execute: three `std::set<std::string>` builds + vector alloc, per einsum, per iteration.
- Fix: `string_einsum` overload taking `EinsumIndices const&` (or the three vectors by const-ref + precomputed links). Cache exists; it's just dropped on the floor.
- `StringDispatch.hpp:108-279` generic fallback: O(rank) pointer-identity scans inside the per-element loop (:207-212,:234-262) -> O(rank^2) bookkeeping per element. Precompute flat (stride,pos) arrays before the loops.
- `Operations.hpp:3872-3879`: batched-GEMM executor allocates 3 pointer arrays sized batch_count per execute; strided layout means base+i*stride - precompute offsets once, refresh base on rebind (or use strided-batched BLAS entry).

### 6. Capture-side churn
- `CaptureContext.hpp:157-179`: `get_or_register` falls back to a linear scan of the whole tensor map per first-seen pre-registered tensor; `_ptr_to_id` cleared each `begin_capture` -> O(T^2) captures. Fix: persistent `void*->TensorId` reverse index on Graph (also removes identical scans at `Graph.cpp:1004,1196` and in `declare_*` at :760,841,880).
- `Node.hpp:228-283`: `inputs/outputs` are heap vectors nearly always holding 1-3 IDs -> small-vector/inline storage; 5 mostly-empty std::functions + large OpData variant bloat Node and hurt scan locality (move async lambdas behind a unique_ptr side struct).
- `TensorHandle.hpp:232-423`: `make_handle` builds 8-10 std::function closures (MPI/GPU/symmetry) per registered tensor even for plain CPU matmuls. Lazily synthesize the rare ones.
- Duplicate dependency edges: `Graph.cpp:788-818` pushes the same (u,v) edge per duplicate read; dedupe when wiring.
- `resolve_alias`/`resolve_owner` byte-identical lambda duplication + per-ID chain walk each sort (`Graph.cpp:420,765`); memoize id->owner.
- `to_json()` (O(N^2)-ish edge reconstruction) runs on every `unregister_graph` (`Graph.cpp:2110-2118`) even with no profiler attached; gate it.

### 7. Parallel-executor timing lock
`src/Executor.cpp:48-56`: every node takes a global `timing_mutex` and push_backs into a shared vector (realloc under lock). Fix: per-worker buffers merged after the wait; reserve up front.

## Tier 2 - machinery that exists but is disconnected

- **MemoryPlanning** (`src/Passes/MemoryPlanning.cpp:141-158`): computes exact liveness intervals + peak/total, stores only report counters. Emit buffer-pool offsets; executor allocates one arena and hands out offsets (the "Potential savings" line it prints is exactly this).
- **InplaceOptimization** (`src/Passes/InplaceOptimization.cpp:64-77`): counts wc==1&&rc==1 candidates, never rewrites. Act on them via existing `Graph::redirect_slot`; extend coverage to "input dies at this node, shape/dtype match" (liveness logic already in FreeInsertion.cpp:96-121).
- **StreamAssignment** (`src/Passes/StreamAssignment.cpp:13-35`): sets `stream_id` no executor reads. Honor it in the Dataflow async path (async_start/finish already exists, Executor.cpp:181-201) for H2D(N+1)/compute(N) overlap.
- **HardwareProfile calibration never loaded**: `detect_default()` (`HardwareProfile.cpp:260-263`) only uses the built-in table; nothing calls `load_json` at runtime. Overlay from `EINSUMS_HARDWARE_PROFILE` env/well-known path when present.
- **calibrate_hardware CLI is dead code**: `tools/calibrate_hardware.cpp:213-216` never calls `parse_args`; `--output/--dtype/...` silently ignored (help advertises them). Thread argc/argv through.
- **ChainParenthesization** duplicates ContractionPlanning's matrix-chain DP with a FLOP-only cost model, analysis-only, unregistered (`ChainParenthesization.cpp:194,238`). Fold or delete.
- **ContractionPlanning runs 27th** (`Optimizer.cpp:256`) - after Materialization(12)/GPUPlacement(14)/MemoryPlanning(25)/FreeInsertion(24). Its restructured GEMMs+intermediates get no placement/distribution/memory management, and it mints eager `create_tensor_dynamic` intermediates (:236) that leak for the graph lifetime (compounds Tier-0 E). Move restructuring into an early planning phase; allocate via `declare_tensor`.

## Tier 3 - transparent pass/coverage extensions

- **ElementWiseFusion** (`ElementWiseFusion.cpp:59-66`): composes `exec_i(); exec_j();` - still two scal sweeps; the `desc_i->factor *=` edit is cosmetic (Scale captured its factor by value, `Operations.hpp:96,127`). Emit one scal with the product; extend to scale->axpy/axpby coefficients, consecutive element_transform, permute-of-permute.
- **Fixed-point cleanup cluster**: `PassManager::run` (`Optimizer.cpp:122`) is one linear sweep; per-pass `modified` bools already exist. Bounded repeat of {ConstantFolding, ScaleAbsorption, PermuteFusion, CSE, DNE, ElementWiseFusion} until quiescent.
- **CSE is O(N^2..N^3)** (`CSE.cpp:152-161` pairwise, :219-229 inner stability scan). Bucket by hash of (kind, redirected inputs, op_data) -> near-linear at CCSD scale.
- **ScaleAbsorption coverage**: absorb input-operand scales into `ab_prefactor` (alpha), BatchedGemm alpha/beta, Axpy/Axpby coefficients; scan beyond the single next writer.
- **GEMMBatching**: pointer-array `gemm_batch` path (:227-250) doesn't require uniform ld, but :170-175 rejects groups on any lda/ldb/ldc mismatch - relax for the pointer path (keep uniform check for strided/GPU); add profile-gated profitability test; currently skips conjugated einsums (:127).
- **Reorder** tie-breaks on bytes_freed only (`Reorder.cpp:117-122`); thread the already-constructed shared HardwareProfile in (as GPUPlacement/ContractionPlanning already do, `Optimizer.cpp:234,256`).
- **ConstantFolding** skips non-Materialized tensors (`ConstantFolding.cpp:106`) and runs pre-Materialization -> most deferred intermediates never fold. Run a second instance post-Materialization; add algebraic identities (scale-by-1, add-zero, identity-permute).
- **Tiled runtime einsum fully serial** (`Detail/TiledRuntimeEinsum.hpp:140-162`, documented at :47): group contributions by output tile coord, dispatch groups across TaskPool, serialize only within a group. Same for `TiledRuntimeElementwise.hpp:35-40` and tiled_axpy/scale/dot/norm.
- **IOPrefetch**: last-writer map rebuilt per DiskRead inside the loop (`IOPrefetch.cpp:80-85`) - build once; prefetch is always max-hoist with no memory-pressure cap - bound hoist distance by an overlap window from HardwareProfile.
- **TransferInsertion** O(N^2) (`TransferInsertion.cpp:49-58,162`); adopt TransferElimination's prebuilt next-use map (`TransferElimination.cpp:27-37`) - and share that builder between the two passes.
- **HardwareProfile lookups**: `find_best_match` re-normalizes static patterns per lookup (`HardwareProfile.cpp:63-80`); `estimate_gemm_gflops` linear scan + std::log per point (`HardwareProfile.hpp:69-87`) - fine now, pre-sort if calibration grows the table.
- **Eager einsum re-parses spec string per call** (`Operations.hpp:3572` -> `EinsumSpec.cpp:70-135`) even for consteval-checked literals; thread-local parse cache or push more into `EinsumFormatString` at consteval.
- **Interned index strings**: a/b/c indices are `vector<string>` copied through EinsumDescriptor/EinsumIndices/ParsedEinsumSpec and re-joined in execute/JSON/DOT; intern to small integer IDs.
- **Executor node hooks**: no started/finished callback point in `Executor.hpp:57-72`; an optional internal hook vector enables NVTX/counters without editing executors.
- **Shared def-use index**: topological_sort, to_json, CSE, MemoryPlanning, FreeInsertion, PermuteFusion each rebuild writer/reader maps from scratch; one graph-owned lazily-built structure with invalidation serves all.

## Benchmark gap

`tests/performance/` contains exactly one benchmark (BenchmarkExecutors.cpp, 284 lines). Capture cost, pass-pipeline cost (`graph.apply`), and replay overhead are unbenchmarked - none of the Tier-1 regressions would be caught today. Add: capture-N-ops, apply-default-pipeline, execute-replay (serial + dataflow) micro-benchmarks.

## Suggested attack order

1. Tier 0 A-D (correctness first: ScaleAbsorption rewrite, CSE redirect indirection, update_prefactors handle, ODR dedupe).
2. `topological_sort` early-out + `effective_io` no-copy (single biggest cheap win; enables measuring the rest honestly).
3. Compiled replay plan (serial first: cached profiler payloads + timing reserve; then Dataflow task-scaffold reuse + batched submission).
4. Wire up the disconnected machinery (InplaceOptimization acting, MemoryPlanning arena, calibration loading, ContractionPlanning phase move).
5. Coverage extensions + benchmarks as they become measurable.

Two agents independently converged on the profiler-string churn and the batched-GEMM pointer arrays.
